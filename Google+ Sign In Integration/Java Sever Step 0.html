Getting started

The application that you will be creating in this codelab will use Google+ and your social graph to let you notify your social graph that you are free.

The Java server runs on top of app engine and requires a Google API project that has the Google+ service enabled. Let's get started!

Requirements

This codelab has the following requirements in order to ease setup and development of the sample.

  • Java 1.6 or higher
  • Eclipse IDE for Java Developers
  • Google Plugin for Eclipse
  • Google App Engine SDK for Java.

Step 1: Create a Google App Engine Project

You can use the Google App Engine SDK to execute the sample application on a local development server.

  1. Navigate to the Google App Engine Admin Console.
  2. Click Create Application.
  3. Choose an application identifier for your app:
    • Type an unique Application Identifier into the text box.
    • Verify that the identifier is available by clicking Check Availability. If not available, try a new identifier.
    • Note or copy the Application Identifier for use in a later step.
  4. Click Create Application.

Step 2: Enable the Google+ API

You need to enable the Google+ API for your app. You can do this by creating a project for your app in the Google APIs Console.

To create an APIs Console project, create an OAuth 2.0 client ID, and register your JavaScript origins and redirect URIs:

  1. In the Google APIs Console, select Create from the pull-down menu on the left, and enter a project name (such as "Sample").
  2. In the Services pane, enable the Google+ API and any other APIs that your app requires.

  3. In the API Access pane, click Create an OAuth 2.0 Client ID.

    • In the Product name field, enter a name for your application (such as "Sample"), and click Next. Providing a product logo is optional.

    • In the Client ID Settings section, do the following:

      • Select Web application for the Application type.
      • Click the more options link.
      • In the Authorized Redirect URIs field, delete the example URI.

      • In the Authorized JavaScript Origins field, add the following URLs.

        http://localhost:8888
        http://<YOUR-APP-ID>.appspot.com
        https://<YOUR-APP-ID>.appspot.com

        (Replace YOUR_APP_ID with the identifier for your app.)

    • Click the Create client ID button.

  4. In the API Access pane, locate the section Client ID for web applications and note or copy the Client ID and Client secret that you will need later to run the sample

Step 3: Setup the application

Download and extract a ZIP file of the project.

  1. Open Eclipse and switch perspectives to use the Java perspective: Window ->Open perspective->Other and choose Java.
  2. Select File->Import and choose General->Existing projects into workspace.
  3. Choose Select root directory and browse to the <path to your unzip directory>/java-server directory. The Free Java project is selected.
  4. Click Finish.
  5. Notice that your project was imported but it has errors.
  6. Select Project->Properties. In the Properties dialog, navigate to Google-> AppEngine and check Use Google App Engine. Click OK. This copies the Google App Engine jars and creates the configuration for the installed version of Google App Engine.
  7. Edit the war/WEB-INF/appengine-web.xml file, insert your Google App Engine application identifier that you created in step 1 in the <application> element, and save the file.
  8. Run the project by selecting Run->Run as->Web application. If you do not see this option, open Run->Run configurations and click Apply without making any changes to force the option to appear.
  9. Navigate to http://localhost:8888 to try the app.

Congratulations! At this point you should have the skeleton for the app running as your server!
