Getting started

The application that you will be creating in this codelab will use Google+ and your social graph to let you notify if you are free.

The Android client runs uses a Free server running on app engine and requires a Google API project that has the Google+ service enabled. Let's get started!

Requirements

This codelab has the following requirements in order to ease setup and development of the sample.

  • A backend Free web service configured and running for the clients to send requests. You should have set this up as the first part of the codelab.
  • Your Free clients and server must define their OAuth 2.0 client settings within the same Google APIs Console project for all features to work correctly.
  • A physical device to use for developing and testing because Google Play services cannot be installed on an emulator.
  • The latest version of the Android SDK, including the SDK Tools component. The SDK is available from the Android SDK Manager.
  • Eclipse configured to use Java 1.6
  • The Google Play services SDK:
    • Launch Eclipse and select Window > Android SDK Manager or run android from the command line.
    • Scroll to the bottom of the package list and select Extras > Google Play services. The package is downloaded to your computer and installed in your SDK environment at /extras/google/google_play_services.
  • The ActionBarSherlock library.

Step 1: Set up a Free backend server

Follow the instructions for setting up a backend server and ensure that you can see the user interface.

Step 2: Add an Android Client ID to your project

You need to add a client ID to your existing project that will be used by your android client. You can do this by navigating to 'API Access' for your project with the Google APIs Console.

  1. In the API Access pane, create an OAuth 2.0 client ID by clicking Create another client ID.
  2. Choose Installed application as your Application type and select Android as type.

  3. In the Package name field, enter your Android's app's package name. The package name used by the sample app is:

    com.google.plus.samples.free

  4. In a terminal, run the the Keytool utility to get the SHA-1 fingerprint of the certificate.

    keytool -exportcert -alias androiddebugkey -keystore <path-to-debug-or-production-keystore> -list -v

    Note: For Eclipse, the debug keystore is typically located at ~/.android/debug.keystore.

  5. The keytool prompts you to enter a password for the keystore. The default password for the debug keystore is android. Keytool then prints the fingerprint to the shell. For example:

    Certificate fingerprint:
      SHA1: DA:39:A3:EE:5E:6B:4B:0D:32:55:BF:EF:95:60:18:90:AF:D8:07:09

  6. Paste the SHA-1 fingerprint into the Signing certificate fingerprint field and click the Create client ID button.

  7. To activate interactive posts, enable the Deep Linking option.

Step 3: Setup the project in Eclipse

Download and extract a ZIP file of the project.

  1. Download ActionBarSherlock library and unzip it.
  2. Launch Eclipse.
  3. Select File > Import > Android > Existing Android Code into Workspace and click Next.
  4. Select Browse.... Enter the path to your Free Android client directory.
    • Import the ActionBarSherlock library library project:
    • Select File > Import > Android > Existing Android Code into Workspace and click Next.
    • Click Browse.... Enter /actionbarsherlock/.
    • Click Finish to import.
  5. Import the Google Play services library project.
    • Select File > Import > Android > Existing Android Code into Workspace and click Next.
    • Select Browse.... Enter /extras/google/google_play_services/.
    • Select google-play-services_lib. Click Finish to import.
  6. Update the Free project properties:
    • Click Project > Properties. The project properties dialog displays.
    • Select Java Build Path, and click the Libraries tab.
    • Click Add External JARs, browse to /extras/android/support/v4/android-support-v4.jar. Click OK.
    • Select Android in the left navigation.
    • Click Add... and select actionbarsherlock to add the ActionBarSherlock library project dependency.
    • Click Add... and select google-play-service_lib to add the Google Play services library project dependency.
  7. Click OK.
  8. Edit the src/config.properties file and modify the api_host property to be the hostname of a Free server.
  9. Connect your Android device by USB to your system.
  10. Select Run > Run to test the Free sample app. Choose Android application if prompted. The sample application will launch on your device.
